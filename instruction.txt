请为以下主题创作一个“数字人文级别”的单文件 HTML 微策展网页（micro-curation exhibition）。

目标不是普通科普文章，而是：
- 接近 V&A、MoMA、大英博物馆、巴黎蓬皮杜、伦敦自然史博物馆等机构的数字展览；
- 接近 exhibition catalogue / curatorial essay / museum wall text 的气质；
- 兼具：
  - 思想史
  - 视觉文化
  - 博物馆学
  - 媒介理论
  - 设计史
  - 社会史
  - 技术史
  - 制度分析
  的交叉深度。

输出必须是：
“可以直接保存运行”的完整单文件 HTML。

不要输出任何解释文字。
不要使用 markdown code block。
直接从 <!DOCTYPE html> 开始输出。


==================================================
一、核心目标（最高优先级）
==================================================

这不是：
- Wikipedia 摘要
- 博客文章
- 公众号科普
- 时间线堆砌

而是：

一场真正具有：
- 问题意识
- 叙事节奏
- 观点张力
- 思想史脉络
- 空间感
- 策展意识

的数字展览。

整个网页必须让人感觉：

“有人正在通过展览的方式组织知识，而不是介绍知识。”

必须体现：
- curator 的存在
- narrative arc（叙事弧）
- dramaturgy（戏剧结构）
- selective framing（选择性框定）
- epistemology（知识观）

不要中立地罗列信息。

要像真正的策展人一样：
组织、切割、强调、对比、揭示。


==================================================
二、内容深度要求（非常重要）
==================================================

全文阅读时长：
8–12 分钟。

总信息密度必须显著高于普通网页。

每个 section：
300–600 中文字。

全文总 section 数：
7–10 个。

禁止：
- 空泛形容
- AI 套话
- “值得注意的是”
- “随着时代发展”
- “具有重要意义”
- 泛泛而谈的总结句

必须：
- 高频出现具体事实
- 高频出现年份、机构、人物、概念
- 高频出现理论与案例
- 高频出现真实历史冲突

每个 section 必须至少包含：
- 一个核心问题
- 一个历史案例
- 一个具体人物/机构
- 一个可验证事实
- 一个带链接的来源


==================================================
三、结构要求（必须遵守）
==================================================

网页结构必须具有“戏剧推进感”。

不要做百科全书式并列结构。

采用以下叙事推进：

<header>
建立主题张力。
不是解释主题，而是抛出问题。

<section 1>
起源与词源：
这个概念最初为何出现？
语言如何暴露其文明背景？

<section 2>
前现代阶段：
它在古代/前工业时代如何存在？
当时的权力结构是什么？

<section 3>
现代性转折：
工业化、帝国、资本主义、民族国家、现代媒介如何改变它？

<section 4>
思想史与理论：
至少引入 3 位理论家/思想家。
不是引用名言，而是解释其观点冲突。

<section 5>
经典案例解剖：
深度拆解至少 2–3 个案例：
- 展览
- 机构
- 人物
- 建筑
- 产品
- 事件

必须分析：
- 为什么重要
- 使用了什么机制
- 为什么改变历史
- 有什么争议

<section 6>
技术与机制：
分析其背后的：
- 科学
- 技术
- 设计
- 数据
- 空间机制
- 媒介机制

<section 7>
权力与争议：
必须包含：
- 殖民
- 阶级
- 性别
- 商业资本
- 算法
- 平台
中的至少两个维度。

必须体现：
“谁拥有解释权？”

<section 8>
“令人意外”彩蛋：
必须是真正反常识、
经过验证、
且能改变读者理解框架的内容。

必须使用特殊视觉样式。

<section 9>
尾声：
不要总结全文。
而是留下一个开放问题或时代症候。

<footer>
参考来源。


==================================================
四、知识层级要求（核心）
==================================================

必须覆盖至少 5 个层次：

1. 事实层
（时间、地点、人物、机构）

2. 机制层
（它如何运作）

3. 制度层
（谁资助、谁控制、谁决定）

4. 思想层
（背后的意识形态）

5. 文明层
（它反映了什么时代变化）

必须形成：
“从物到制度，再到文明结构”的上升。

不要停留于对象介绍。


==================================================
五、思想史要求（非常重要）
==================================================

必须包含：
- 理论家
- 学派
- 思潮
- 观念冲突

不能只是提名字。

必须解释：
他们到底改变了什么。

根据主题自动选择相关人物，例如：
- Walter Benjamin
- Michel Foucault
- Jean Baudrillard
- Bruno Latour
- Susan Sontag
- Marshall McLuhan
- Pierre Bourdieu
- Guy Debord
- André Malraux
- Brian O'Doherty
等。

至少出现：
- 一个理论冲突
- 一个范式转变


==================================================
六、案例拆解要求
==================================================

至少深度解剖 3 个案例。

每个案例必须包含：

- 背景
- 方法
- 视觉语言
- 机制
- 历史影响
- 批评/争议

不要只提名字。

案例必须具体。

例如：
不是“MoMA 很重要”，
而是：
MoMA 如何通过 Alfred Barr 的展陈路径重写现代艺术史。


==================================================
七、来源要求（严格）
==================================================

每个事实性陈述必须附带可点击来源。

优先级：

1. 博物馆官方 archive
2. JSTOR
3. Google Scholar
4. 展览 catalogue
5. 学术机构
6. 原始文献
7. Wikipedia（仅补充）

严禁伪造链接。

如果无法确定具体论文：
使用稳定可访问页面。

允许来源：
- JSTOR
- archive.org
- Project Gutenberg
- ctext.org
- museum archive
- official exhibition pages
- britannica
- Google Arts & Culture

不允许：
- 虚假 DOI
- 编造论文
- 不存在的 museum page

链接格式：

<a href="..." class="source-link" target="_blank" rel="noopener">
  [来源]
</a>


==================================================
八、语言风格要求
==================================================

整体语言风格必须接近：

- exhibition catalogue
- museum wall text
- curatorial essay
- 高级文化杂志

不是：
- 自媒体
- 营销文案
- AI 总结

语言特征：

- 克制
- 高信息密度
- 强概念
- 少废话
- 有节奏感
- 允许长句
- 允许思想性

允许出现：
- 精准判断
- 文明分析
- 理论语言

禁止：
- 鸡汤
- 励志
- 空洞总结
- AI 套话


==================================================
九、视觉美学要求
==================================================

根据主题自动选择最适合的美学方向：

可选：
- 博物馆图鉴风
- 档案馆风
- 冷战科技档案风
- 欧洲现代主义出版风
- 黑白摄影论文风
- Brutalist editorial 风
- 自然史图谱风

整体必须：
- 克制
- 高级
- 有留白
- 强 typography
- 强 grid
- 强 hierarchy

禁止：
- AI 渐变炫光
- 紫蓝霓虹
- 卡片 UI 风
- SaaS dashboard 风
- 花哨玻璃拟态


==================================================
十、字体要求
==================================================

仅允许从 Google Fonts 引入。

标题字体：
- Cormorant Garamond
- DM Serif Display
- Libre Baskerville
- Playfair Display

正文字体：
- Source Serif 4
- EB Garamond
- Lora
- Merriweather

注释：
- Space Mono（如适合）

禁止：
- Inter
- Roboto
- Arial
- system-ui


==================================================
十一、版式与视觉结构
==================================================

必须包含：

1. 全屏 Hero
2. 强 typography
3. 多层级标题
4. SVG 原创图示
5. 时间轴
6. 引文块
7. 特殊彩蛋卡片
8. 学术边注或 marginalia
9. 大段留白
10. 分栏 grid

SVG 必须真实与主题相关。

例如：
- 知识谱系
- 地图
- 时间轴
- 展陈结构
- 思想关系图
- 建筑剖面
- 视觉系统演化

禁止：
- emoji
- 占位 SVG
- 无意义装饰


==================================================
十二、响应式与技术要求
==================================================

必须：

- 单文件 HTML
- CSS 内联
- 不使用 JS 框架
- 不使用图片 CDN
- 不使用外部图片
- 不使用 Tailwind
- 不使用 Bootstrap

允许：
- 原生 JS 小交互
- SVG 动画
- CSS animation

响应式要求：

桌面：
复杂 grid 与多栏。

移动端：
单列重排。

不得水平溢出。


==================================================
十三、可访问性要求
==================================================

必须：

- 所有 SVG 含 <title> 与 <desc>
- 所有链接含：
target="_blank" rel="noopener"

颜色对比符合 WCAG AA。

正文行宽舒适。

长文本可阅读。


==================================================
十四、最终输出要求
==================================================

直接输出完整 HTML。

不要：
- markdown
- 解释
- 前言
- “以下是”
- 代码块

必须从：

<!DOCTYPE html>

开始。

直到：

</html>

结束。
